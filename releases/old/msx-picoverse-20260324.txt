multirom.pio: add LRU bank cache and AMD flash emulation for ASCII16-X/NEO8/NEO16

Port the mapper-aware LRU bank cache system from loadrom.pio to
multirom.pio. All three large-ROM mappers now serve reads exclusively
from SRAM instead of flash XIP, matching loadrom.pio behaviour:

- ASCII16-X: 12-slot (16KB) LRU cache with AMD flash command emulation
  (byte-program, sector-erase) for cartridge detection and save data
- NEO8: 24-slot (8KB) LRU cache with cached write handler
- NEO16: 12-slot (16KB) LRU cache with cached write handler

The bank cache divides the 192KB SRAM pool into fixed-size slots and
uses LRU eviction with pinned active pages. Flash is only accessed on
cache misses during bank switches. Bank numbers are normalized modulo
ROM size to support hardware mirroring.

Sunrise/Nextor implementations verified identical across both firmwares.
